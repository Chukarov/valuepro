КАК ДА УПРАВЛЯВАТЕ СЪДЪРЖАНИЕТО — Valoris Asset
===============================================
Съдържанието вече е разделено на файлове по раздели в папка content/.
Промените се публикуват АВТОМАТИЧНО (GitHub Action: build.py → dist → Pages).

ФАЙЛОВЕ
-------
content/ ............ съдържанието по раздели (всеки файл = един раздел/страница):
   site.json ......... фирмени данни (име, телефон, имейл)
   navigation.json ... текстове в менюто и футъра
   home.json ......... начална страница
   services.json ..... 6-те страници с услуги
   pricing.json ...... ценоразпис (вкл. редовете с цени)
   about.json ........ За нас / Методология / Мрежа
   clients.json ...... Клиенти
   blog.json ......... Блог
   faq.json .......... Въпроси
   contact.json ...... Контакти
   misc.json ......... 404 страница
build.py ............ генератор (рядко се пипа)
assets/ ............. CSS, JS, лого
admin/ .............. Sveltia CMS панел
.github/workflows/ .. авто-публикуване
dist/ ............... РЕЗУЛТАТ (създава се автоматично — не качвайте)

НАЧИН 1 — РЕДАКТИРАНЕ НА ФАЙЛ В content/ (без настройка)
-------------------------------------------------------
Отворете напр. content/contact.json в GitHub, сменете текста между кавичките
(български = "bg", английски = "en"), commit. Сайтът се преизгражда за 1–2 мин.
Не пипайте имената на ключовете; пазете кавичките/запетаите (проверка: jsonlint.com).

НАЧИН 2 — SVELTIA CMS ПАНЕЛ (визуален, с логин)
-----------------------------------------------
ЕДНОКРАТНА НАСТРОЙКА
  1. В admin/config.yml сменете   repo: OWNER/REPO   с вашето repo
     (напр. repo: ivan/valoris-asset). Commit.
ВХОД (без сървър — токен)
  2. Отворете https://ВАШИЯТ-САЙТ/admin/
  3. Натиснете „Sign In with Token".
  4. Диалогът дава връзка към GitHub за създаване на токен (с готови права) —
     създайте го (изберете срок, напр. 90 дни), копирайте и поставете обратно.
  5. Влизате. Готово.
РЕДАКЦИЯ
  6. Вляво има 11 раздела (Company info, Home page, Pricing, Contact, ...).
     Изберете раздел → разгънете секциите (bg / en) → сменете полетата.
  7. Натиснете Save / Publish. Записва се в съответния content/*.json →
     Action-ът преизгражда и публикува сайта.
ЗАБЕЛЕЖКА: токенът изтича (по подразбиране ~90 дни). Когато входът спре да работи,
повторете стъпки 3–5 с нов токен. (За вход без изтичащ токен — безплатен Cloudflare
Worker: github.com/sveltia/sveltia-cms-auth — нужен само ако ще ползвате панела хора,
които не са технически.)

ПУБЛИКУВАНЕ В GITHUB PAGES (еднократно)
---------------------------------------
Settings → Pages → Source: GitHub Actions  (НЕ „Deploy from a branch").
Качете: content/, build.py, assets/, admin/, .github/, .htaccess,
site.webmanifest, HOW-TO-EDIT.txt. (НЕ качвайте dist/.)
